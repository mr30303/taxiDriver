# 아키텍처 의사결정

## 2026-02-26
- 사용자 기능으로서 "화장실 추가"는 당분간 사용하지 않습니다.
- 이메일 인증은 보류 상태를 유지합니다.
- 회원가입 시 닉네임 입력을 필수로 두고, 댓글 작성자 표시는 닉네임 우선으로 합니다.
- 지도 진입 시 현재 위치로 자동 이동하는 카메라 동작은 제거합니다.
- 커뮤니티 화면은 한글 UI를 유지하고, 화장실 중심 요약 카드 구조를 사용합니다.
- 커뮤니티 카드 탭 시 지도 화면으로 이동하고 대상 화장실 상세를 포커스/오픈합니다.
- 댓글 소유자 검증은 다중 레이어로 적용합니다.
  - UI/ViewModel 소유자 확인
  - 저장소에서 수정/삭제 전 소유자 재검증
  - Firestore 규칙의 `request.auth.uid` 기준 검증
- 화장실 숨기기는 서버 권한 거부 시에도 로컬 상태 폴백으로 사용자 경험을 유지합니다.
- 브랜딩 방향은 흰색 런처 배경 + 45도 택시 스타일 아이콘으로 확정합니다.
- 외부 설치/테스트를 위해 서명된 릴리즈 산출물을 생성/공유합니다.
